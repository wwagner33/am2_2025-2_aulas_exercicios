1. Introdução: O que é CSS?

Em termos simples: O CSS (Cascading Style Sheets) é a linguagem que usamos para "vestir" o HTML.

  • Pense no HTML como o esqueleto de uma página web: ele define a estrutura e o conteúdo (parágrafos, títulos, imagens).
  • Pense no CSS como a pele, as roupas e a maquiagem: ele define a aparência, o layout, as cores, as fontes e as animações.

O que significa "Cascading"?

O nome "Folhas de Estilo em Cascata" é importante:

  • Folhas de Estilo: O código CSS é escrito em "folhas" de regras de estilo.
  • Cascata: Este é o conceito mais crucial. O navegador decide qual regra de estilo aplicar a um elemento com base em uma ordem de prioridade (a "cascata"). Regras mais específicas (como um ID) geralmente substituem regras mais gerais (como uma tag).

2. Uma Breve História

A necessidade do CSS surgiu da "bagunça" do HTML inicial.

  • O Problema (Início dos anos 90): O HTML não foi feito para design. Tags como <font> e <center> misturavam estrutura e estilo, tornando a manutenção um pesadelo.
  • A Proposta (1994): Håkon Wium Lie propôs a ideia de "Folhas de Estilo" para separar o conteúdo da apresentação.
  • CSS 1 (1996): O W3C publicou a primeira recomendação. Era básica: cobria fontes, cores e margens.
  • CSS 2 (1998): Introduziu conceitos poderosos como position (absolute, relative, fixed) e z-index.
  • CSS 3 (De 1999 até hoje): A grande revolução. O CSS foi dividido em módulos (Selectors, Colors, Animations, Grid Layout). É por isso que não falamos em "CSS 4". Recursos como Flexbox e Grid são módulos do CSS3.

3. O Tijolo Fundamental: O "Box Model"

Para o CSS, todo elemento HTML é uma caixa retangular. O "Box Model" (Modelo de Caixa) define como essa caixa é construída, de dentro para fora:

  1. Content (Conteúdo): A área do seu texto e imagens (definida por width e height).
  2. Padding (Preenchimento): O espaço interno transparente, entre o conteúdo e a borda.
  3. Border (Borda): A linha que fica ao redor do padding.
  4. Margin (Margem): O espaço externo transparente, fora da borda. Ele "empurra" outros elementos.
.minha-caixa {
  /* 1. Conteúdo */
  width: 200px; 
  height: 100px;

  /* 2. Preenchimento (espaço interno) */
  padding: 20px; 

  /* 3. Borda (linha ao redor) */
  border: 5px solid black; 
  
  /* 4. Margem (espaço externo) */
  margin: 30px; 
}

4. Definindo Tamanhos: Unidades de Medida

Como definimos o tamanho das caixas? Usamos unidades de medida.

📏 Unidades Absolutas

São unidades de medida fixas. Seus valores não mudam.

  • px (Pixel): A unidade absoluta mais comum. É um "ponto" na tela. É inflexível e não se adapta bem à acessibilidade ou responsividade.
  • Outras (cm, mm, in, pt): Raramente usadas, exceto para folhas de estilo de impressão (@media print).

🔄 Unidades Relativas

São unidades flexíveis e dinâmicas, calculadas com base em outro valor. São a base do design responsivo.

  • % (Porcentagem): Relativa ao elemento pai. Se o pai tem 500px de largura, um filho com width: 50% terá 250px.
  • rem (Root EM): Relativa ao tamanho da fonte do elemento raiz (a tag <html>). Esta é a melhor unidade moderna para font-size, margin e padding.

4. Unidades Relativas (Continuação)

  • vh (Viewport Height): 1% da altura da janela do navegador. height: 100vh força um elemento a ter exatamente a altura da tela.
  • vw (Viewport Width): 1% da largura da janela do navegador.

Quando usar o quê?

Unidade Quando Usar (Regra Geral)
px Para bordas finas (border-width: 1px) ou tamanhos fixos que nunca devem mudar.
rem A melhor escolha para font-size, margin e padding. Garante acessibilidade.
% Ideal para larguras de layout (ex: colunas width: 50%) dentro de um container.
vh / vw Para elementos que devem se dimensionar em relação à tela inteira (ex: banners, seções de tela cheia).

5. Controlando o Layout: A Propriedade position

A propriedade position nos permite quebrar o "fluxo normal" da página e controlar exatamente onde um elemento fica. Ela "desbloqueia" o uso de top, right, bottom e left.

static (Padrão)

O elemento segue o fluxo normal. As propriedades top, left, etc., são ignoradas.

relative

O elemento é posicionado relativo à sua posição original.

  • Ele ainda ocupa seu espaço original no fluxo (os outros elementos não se movem para ocupar seu lugar).
  • top: 20px; move o elemento 20px para baixo de onde ele estaria.
  • Uso principal: Servir de "ponto de âncora" para elementos absolute.

absolute

O elemento é removido do fluxo normal (os outros elementos agem como se ele não existisse).

  • Ele é posicionado em relação ao ancestral posicionado mais próximo (qualquer "pai" ou "avô" que tenha position diferente de static).

5. position (Cont.) e z-index

fixed

O elemento é removido do fluxo normal. Ele é posicionado em relação à janela do navegador (viewport) e não se move com a rolagem. Perfeito para menus fixos.

sticky

Um híbrido de relative e fixed. O elemento age como relative até que a rolagem o faça atingir um limite (ex: top: 0;), então ele "gruda" e se comporta como fixed.

6. Controlando a Profundidade: z-index

O z-index controla a ordem de empilhamento (qual elemento fica "por cima" de qual).

  • A Regra de Ouro: z-index só funciona em elementos com position diferente de static.
  • Valores são números inteiros. Maior número = mais na frente.

7. Galeria de Exemplos (1/2)

Exemplo 1: Centralizar com Flexbox

O display: flex é a forma moderna de alinhar itens.

.container {
  display: flex;
  justify-content: center; /* Centraliza horizontalmente */
  align-items: center; /* Centraliza verticalmente */
  height: 100vh;
}

Exemplo 2: Layout com CSS Grid

O display: grid permite criar layouts complexos de linhas e colunas.

.grid-container {
  display: grid;
  /* "Desenha" o layout com texto */
  grid-template-areas:
    "header header"
    "nav    main"
    "footer footer";
  grid-template-columns: 1fr 3fr; /* Coluna 1 = 1 fração, Coluna 2 = 3 frações */
}

7. Galeria de Exemplos (2/2)

Exemplo 3: Card Responsivo (Mobile-First)

.card {
  width: 90%; /* Em telas pequenas, ocupa 90% */
  max-width: 350px; /* Em telas grandes, não passa de 350px */
  margin: 20px auto; /* Centraliza o card */
  box-shadow: 0 4px 8px rgba(0,0,0,0.1);
}

Exemplo 4: Tema Claro / Escuro (Variáveis CSS)

/* 1. Define as variáveis padrão (tema claro) */
:root {
  --bg-color: #ffffff;
  --text-color: #333333;
}
/* 2. Sobrescreve se o usuário preferir tema escuro */
@media (prefers-color-scheme: dark) {
  :root {
    --bg-color: #1a1a1a;
    --text-color: #f0f0f0;
  }
}
/* 3. Aplica as variáveis */
body {
  background-color: var(--bg-color);
  color: var(--text-color);
}

9. Próximo Nível e Bibliografia

Os conceitos de Box Model e Position são a base. O próximo passo é dominar os módulos de layout modernos:

  • Flexbox (Flexible Box Layout): Para layout em uma dimensão (linhas ou colunas).
  • CSS Grid (Grid Layout): Para layout em duas dimensões (linhas e colunas).
