Protocolo HTTP: A Comunicação da Web

O HTTP (HyperText Transfer Protocol) é o protocolo fundamental da web. Ele define um conjunto de regras para a comunicação entre um cliente (seu navegador) e um servidor (onde o site está hospedado).

O Ciclo Requisição-Resposta:

  1. Requisição (Request): Seu navegador envia uma requisição HTTP para o servidor, pedindo um recurso (ex: uma página HTML, uma imagem).
  2. Processamento: O servidor recebe a requisição e a processa.
  3. Resposta (Response): O servidor envia de volta uma resposta HTTP contendo o recurso solicitado e um código de status (ex: 200 OK).
3 / 31

Principais Métodos HTTP

Uma requisição HTTP possui um "método" que indica a ação desejada a ser executada no recurso solicitado.

GET

O mais comum. Usado para solicitar (obter) dados de um recurso. Quando você acessa um site, seu navegador faz uma requisição GET para buscar o HTML da página.

POST

Usado para enviar dados para um servidor para criar um novo recurso. Ex: enviar um formulário de cadastro.

PUT

Usado para enviar dados para substituir completamente um recurso existente. Ex: atualizar o perfil de um usuário com novas informações.

DELETE

Usado para remover um recurso específico. Ex: apagar um post em uma rede social.

Engines de Renderização

Uma engine de renderização (ou motor de layout) é o componente principal de um navegador. Sua função é "ler" o código HTML e CSS e transformá-lo na página visual com a qual você interage.

As Três Grandes Engines Atuais:

Blink

Desenvolvida pelo Google. É a engine mais usada no mundo, presente no:

  • Google Chrome
  • Microsoft Edge
  • Opera
  • Brave

Gecko

Desenvolvida pela Mozilla Foundation, é o coração do navegador:

  • Mozilla Firefox

WebKit

Desenvolvida pela Apple, é a base para os navegadores em seus sistemas:

  • Safari (iOS, macOS)

O que é HTML?

HTML (HyperText Markup Language) é a linguagem de marcação que usamos para criar e organizar o conteúdo em uma página da web.

Pense nele como o esqueleto de um site. Ele define os diferentes blocos de conteúdo, como títulos, parágrafos, imagens e links, dando uma estrutura lógica ao documento.

6 / 31

HTML não é uma Linguagem de Programação

HTML (Marcação)

O HTML é uma linguagem declarativa. Você "marca" o conteúdo para dizer ao navegador o que ele é (um título, uma lista, uma imagem), mas não como ele deve se comportar.

  • ✔️ Define a estrutura e o significado.
  • ❌ Não possui lógica condicional (if/else).
  • ❌ Não possui laços de repetição (loops).
  • ❌ Não pode executar cálculos ou manipular dados.

Linguagem de Programação

Uma linguagem de programação, como JavaScript, é usada para criar lógica e comportamento. Ela diz ao computador como fazer as coisas.

  • ✔️ Executa algoritmos e tarefas.
  • ✔️ Usa lógica, loops e funções.
  • ✔️ Responde a eventos do usuário.
  • ✔️ Manipula dados e se comunica com servidores.

Estrutura Básica de um Arquivo HTML5

<!DOCTYPE html>

Informa ao navegador que o documento é HTML5.

<html lang="pt-BR">

Elemento raiz. O atributo `lang` define o idioma da página.

<head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Título da Página</title> </head>

Contém metadados: `charset` para codificação de caracteres, `viewport` para responsividade e `title` para o título da aba.

<body> <!-- Todo o conteúdo visível da página vai aqui --> </body> </html>

Corpo do documento, onde todo o conteúdo visual é inserido.

Organizando Conteúdo: Listas

Listas Não Ordenadas `<ul>`

Usadas quando a ordem dos itens não importa. Geralmente renderizadas com marcadores (bolinhas).

<ul> <li>Café</li> <li>Leite</li> <li>Pão</li> </ul>

Listas Ordenadas `<ol>`

Usadas quando a sequência dos itens é importante. Renderizadas com números ou letras.

<ol> <li>Primeiro, misture os ingredientes.</li> <li>Segundo, asse por 30 minutos.</li> <li>Terceiro, sirva quente.</li> </ol>
9 / 31

Inserindo Imagens `<img>`

A tag `<img>` é usada para incorporar imagens em uma página. Ela é uma tag "vazia", ou seja, não possui fechamento.

<img src="caminho/para/imagem.jpg" alt="Descrição da imagem">

Atributo `src` (Source)

É obrigatório e especifica o caminho (URL) para o arquivo de imagem que será exibido.

Atributo `alt` (Alternative Text)

Fornece um texto alternativo para a imagem. É crucial para a acessibilidade, pois é lido por leitores de tela para usuários com deficiência visual e exibido se a imagem não carregar.

10 / 31

Semântica para Imagens: `<figure>` e `<figcaption>`

Para associar semanticamente uma imagem a uma legenda, o HTML5 oferece as tags `<figure>` e `<figcaption>`.

  • <figure>: "Embrulha" a mídia (imagem, diagrama, etc.) que faz parte do fluxo principal, mas poderia ser movida para outro lugar (como um apêndice) sem afetar o sentido do texto.
  • <figcaption>: Fornece a legenda para o conteúdo dentro do `<figure>`.

Exemplo Prático

<figure> <img src="por-do-sol.jpg" alt="Pôr do sol na praia de Ipanema"> <figcaption>Fig.1 - Pôr do sol visto do Arpoador, Rio de Janeiro.</figcaption> </figure>

Boas Práticas em HTML

Use Tags Semânticas

Utilize tags como <header>, <nav>, <main>, <article> e <footer> em vez de apenas <div>. Isso melhora o SEO e a acessibilidade.

Priorize a Acessibilidade

Sempre use o atributo alt em imagens. Utilize <label> para campos de formulário. A web deve ser para todos.

Mantenha o Código Limpo

Use indentação para mostrar a hierarquia dos elementos. Escreva tags e atributos em letras minúsculas por convenção.

Antes do CSS: A Era do Caos

Nos primórdios da web, não havia separação entre estrutura e estilo. A aparência era definida diretamente no HTML, usando tags e atributos que hoje são obsoletos.

Como era feito:

<body bgcolor="#FFFFFF"> <center> <font color="blue" size="5"> Este é um título azul! </font> </center> </body>

Problemas: Manutenção difícil, código poluído e nenhuma consistência visual entre páginas.

13 / 31

CSS: O Estilo da Página

CSS (Cascading Style Sheets) é a linguagem que usamos para estilizar documentos HTML. Pense nele como o maquiador e designer de interiores de um site.

Ele foi criado para separar completamente a apresentação (cores, fontes, layout) da estrutura (conteúdo).

14 / 31

CSS também não é uma Linguagem de Programação

Assim como o HTML, o CSS é uma linguagem declarativa. Você declara "regras" de estilo que o navegador deve aplicar aos elementos.

  • ✔️ Descreve a aparência dos elementos.
  • ✔️ Pode ter "variáveis" (Custom Properties) e cálculos simples (calc()).
  • ❌ Não possui a lógica computacional completa de uma linguagem de programação.

O CSS determina como algo deve parecer, enquanto uma linguagem de programação determina como algo deve funcionar.

15 / 31

3 Formas de Incluir CSS

1. Externo (Recomendado)

Um arquivo .css separado é linkado no <head>. Permite reutilizar estilos em várias páginas.

<link rel="stylesheet" href="estilos.css">

2. Interno

O código CSS é colocado dentro da tag <style> no <head>. Útil para estilos específicos de uma única página.

<style> body { background-color: #f0f0f0; } </style>

3. Inline (Evitar)

O estilo é aplicado diretamente no elemento através do atributo `style`. Dificulta a manutenção.

<p style="color: blue;">...</p>

CSS: O Modelo de Caixa (Box Model)

Todo elemento HTML pode ser considerado uma caixa. O Box Model define como o tamanho e o espaçamento dessa caixa são calculados.

  • Content: O conteúdo real (texto, imagem).
  • Padding: Espaçamento interno.
  • Border: A borda.
  • Margin: Espaçamento externo.
17 / 31

CSS Avançado: Pseudo-classes e Variáveis

Pseudo-classes

São palavras-chave que definem um estado especial do elemento, permitindo estilização dinâmica.

/* Estilo quando o mouse está sobre o link */ a:hover { text-decoration: underline; } /* Estilo para cada linha ímpar de uma lista */ li:nth-child(odd) { background-color: #eee; }

Variáveis (Custom Properties)

Permitem armazenar valores reutilizáveis, facilitando a manutenção e a criação de temas.

:root { --cor-primaria: #0056b3; } a { color: var(--cor-primaria); }
18 / 31

Boas Práticas em CSS (1/2)

Organização e Nomenclatura

Use uma metodologia de nomenclatura como o BEM (Block, Element, Modifier) para evitar conflitos e tornar seu CSS mais legível.

/* Bloco */ .card {} /* Elemento do Bloco */ .card__title {} /* Modificador do Bloco */ .card--dark {}

Separação de Arquivos

Para projetos grandes, divida seu CSS em múltiplos arquivos por componente (botões, formulários, etc.) e importe-os em um arquivo principal. Isso melhora a organização e a manutenção.

19 / 31

Boas Práticas em CSS (2/2)

Mobile First

Comece estilizando para dispositivos móveis primeiro e, em seguida, use media queries para adaptar o layout para telas maiores. Isso resulta em um design mais robusto e performático.

/* Estilos base (mobile) */ .container { width: 100%; } /* Estilos para telas maiores */ @media (min-width: 768px) { .container { width: 750px; } }

Use Unidades Relativas

Prefira unidades como rem (para fontes e espaçamentos) e % (para layouts) em vez de px. Isso torna seu design mais flexível e acessível, pois se adapta às configurações de fonte do usuário.

HTML5: A Era da Multimídia

Com o HTML5, incorporar áudio e vídeo se tornou nativo. As tags <audio> e <video> oferecem controle total sobre a mídia.

Atributos Comuns

  • controls: Exibe os controles padrão.
  • autoplay: Inicia a reprodução automaticamente.
  • muted: Inicia a mídia sem som.
  • loop: Faz a mídia reiniciar ao terminar.
  • poster: Exibe uma imagem antes do vídeo.

Uso com a tag `<source>`

Para maior compatibilidade, você pode fornecer múltiplos formatos de arquivo.

<video controls poster="capa.jpg"> <source src="video.mp4" type="video/mp4"> <source src="video.webm" type="video/webm"> </video>

O que é JavaScript?

JavaScript (JS) é uma linguagem de programação de alto nível, dinâmica e interpretada, que é uma das tecnologias centrais da World Wide Web.

  • Criador: Brendan Eich na Netscape.
  • Propósito Inicial: Adicionar interatividade e comportamento às páginas web, que antes eram estáticas.
  • Padrão: A especificação da linguagem é chamada de ECMAScript, mantida pela ECMA International (comitê TC39).

Não confunda com Java!

Apesar do nome semelhante (uma decisão de marketing na época), JavaScript e Java são linguagens completamente diferentes, com sintaxe, semântica e casos de uso distintos.

JavaScript: Pontos Positivos e Negativos

👍 Pontos Positivos

  • Onipresente: Roda em todos os navegadores, no servidor (Node.js), em dispositivos móveis e desktops.
  • Grande Ecossistema: Possui uma vasta coleção de bibliotecas (React, Vue) e ferramentas (npm, webpack).
  • Flexível e Dinâmico: A tipagem dinâmica permite prototipagem rápida.
  • Rápido: Engines modernas como a V8 usam compilação Just-In-Time (JIT) para otimizar a performance.

👎 Pontos Negativos

  • Tipagem Dinâmica: Pode levar a erros difíceis de rastrear em aplicações grandes. (Solução: TypeScript).
  • Comportamentos Inesperados: A linguagem possui algumas "pegadinhas" e coerções de tipo que podem confundir iniciantes.
  • Herança Prototípica: O modelo de orientação a objetos pode ser menos intuitivo que o baseado em classes de outras linguagens.

Interpretadores e Ambientes JavaScript

O código JavaScript não é compilado para um executável antes de rodar; ele é interpretado em tempo real por uma Engine JavaScript.

Principais Engines:

V8

Desenvolvida pelo Google, de código aberto e escrita em C++. É a engine do Google Chrome e do ambiente de servidor Node.js.

SpiderMonkey

A primeira engine JavaScript, criada por Brendan Eich. Hoje, é mantida pela Mozilla e usada no Firefox.

JavaScriptCore

Desenvolvida pela Apple para o navegador Safari. Também é usada em outros contextos dentro do ecossistema da Apple.

JavaScript & DOM: A Interatividade

O que eles fazem juntos?

O JavaScript é a linguagem que dá vida à página. Para fazer isso, ele manipula o DOM (Document Object Model).

O DOM é uma representação em árvore de todo o seu código HTML. O JavaScript pode navegar nessa árvore para encontrar, adicionar, remover ou modificar qualquer elemento, tornando a página dinâmica e interativa.
