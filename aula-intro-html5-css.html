A História do HTML

  • 1991: Tim Berners-Lee, no CERN, cria o HTML (HyperText Markup Language) para compartilhar documentos.
  • 1995: HTML 2.0 é a primeira especificação padrão.
  • 1997: HTML 3.2 introduz tabelas, applets e texto flutuante.
  • 1999: HTML 4.01 se torna um padrão, focando na separação de conteúdo e apresentação (com CSS).
  • 2014: HTML5 é finalizado, trazendo uma revolução com suporte nativo a vídeo, áudio, semântica e muito mais.

Protocolo HTTP

O que é?

HTTP (HyperText Transfer Protocol) é o protocolo de comunicação que permite a transferência de dados na World Wide Web. É a base da comunicação entre o seu navegador (cliente) e o servidor onde um site está hospedado.

Como funciona?

  1. O cliente (navegador) envia uma requisição (request) HTTP para o servidor.
  2. O servidor processa a requisição.
  3. O servidor envia de volta uma resposta (response) HTTP com os dados (ex: o código HTML).

Principais Métodos HTTP

  • GET: Solicita a representação de um recurso. Usado para buscar dados (ex: carregar uma página).
  • POST: Envia dados para o servidor para criar um novo recurso (ex: enviar um formulário de cadastro).
  • PUT: Envia dados para o servidor para substituir um recurso existente.
  • DELETE: Remove um recurso específico do servidor.
  • PATCH: Aplica modificações parciais a um recurso.

Códigos de Status HTTP

Sucesso e Redirecionamento

  • 2xx (Sucesso): A requisição foi bem-sucedida. Ex: 200 OK.
  • 3xx (Redirecionamento): Ações adicionais são necessárias. Ex: 301 Moved Permanently.

Erros do Cliente e Servidor

  • 4xx (Erro do Cliente): A requisição contém sintaxe inválida ou não pode ser atendida. Ex: 404 Not Found.
  • 5xx (Erro do Servidor): O servidor falhou em atender uma requisição válida. Ex: 500 Internal Server Error.

Engines de Renderização

São os "motores" dentro dos navegadores que interpretam o código HTML e CSS e o transformam na página visual que interagimos.

Principais Engines

  • Blink: Usada no Google Chrome, Microsoft Edge, Opera e outros.
  • Gecko: Usada no Mozilla Firefox.
  • WebKit: Usada no Apple Safari.

Documento Básico HTML5

<!DOCTYPE html> <html lang="pt-BR"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Título da Página</title> </head> <body> <h1>Olá, Mundo!</h1> <p>Este é um parágrafo.</p> </body> </html>

O que o HTML5 trouxe de novo?

Multimídia e Gráficos

  • Tags <audio> e <video> para incorporar mídias sem plugins.
  • <canvas> para desenhos e animações via JavaScript.
  • <svg> para gráficos vetoriais escaláveis.

Semântica e Estrutura

  • Novas tags que descrevem o conteúdo: <header>, <footer>, <nav>, <article>, <section>.
  • Melhora a acessibilidade e o SEO (Otimização para Mecanismos de Busca).
  • Formulários com novos tipos de input (email, date, etc.).

Principais Tags Semânticas do HTML5

Estruturam a página de forma lógica e significativa.

  • <header>: Cabeçalho da página ou de uma seção.
  • <nav>: Agrupa os links de navegação.
  • <main>: Define o conteúdo principal do documento.
  • <article>: Conteúdo independente e autossuficiente (ex: post de blog).
  • <section>: Uma seção genérica de um documento.
  • <aside>: Conteúdo lateral, relacionado ao conteúdo principal (ex: sidebar).
  • <footer>: Rodapé da página ou de uma seção.

CSS: O Estilista da Web

Por que foi criado?

O HTML original misturava estrutura e aparência (ex: <font color="red">). Isso tornava a manutenção difícil. O CSS (Cascading Style Sheets) foi criado para separar a apresentação da estrutura do documento.

Qual sua utilidade?

Com CSS, você controla cores, fontes, espaçamentos, layout, animações e a responsividade (adaptação a diferentes telas) de um site, mantendo o HTML limpo e focado apenas no conteúdo e em sua estrutura semântica.

3 Formas de Incluir CSS

1. Inline (Atributo `style`)

Aplica o estilo diretamente na tag HTML. Use com moderação.

<p style="color: blue;">Texto azul.</p>

2. Interno (Tag `<style>`)

Dentro do <head> do documento, para estilos daquela página.

<style> p { color: blue; } </style>

3. Externo (Arquivo .css)

A forma mais recomendada. Um arquivo .css linkado no <head>.

<link rel="stylesheet" href="estilos.css">

Anatomia de uma Regra CSS

  • Seletor (Selector): O elemento HTML que você quer estilizar (ex: p, .minha-classe, #meu-id).
  • Propriedade (Property): O aspecto visual que você quer mudar (ex: color, font-size).
  • Valor (Value): A configuração que você quer aplicar à propriedade (ex: blue, 16px).

Tipografia com Google Fonts

Passo a Passo

  1. Acesse fonts.google.com.
  2. Escolha uma fonte (ex: Roboto) e selecione os pesos desejados.
  3. Copie o código <link> fornecido pelo Google e cole no <head> do seu HTML.
  4. No seu CSS, use a propriedade font-family com o nome da fonte.

Exemplo Prático

1. No HTML (dentro do `head`):

<link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">

2. No CSS:

body { font-family: 'Roboto', sans-serif; }

Aplicação Prática: Código dos Cards

Estrutura HTML

Criamos um `checkbox` invisível para controlar o estado (visível/invisível) e o `label` funciona como nosso botão. O card a ser afetado tem a classe `.disappear`.

<!-- O botão que controla --> <input type="checkbox" id="card-toggle" style="display: none;"> <label for="card-toggle">Fazer Card 2 Sumir/Aparecer</label> <!-- O container dos cards --> <div class="card-container"> <div class="card">Card 1</div> <div class="card disappear">Card 2</div> <div class="card">Card 3</div> </div>

Estilização e Lógica CSS

A "mágica" acontece com o seletor `+` (adjacente) e a pseudo-classe `:checked`. Quando o checkbox é marcado, a regra de CSS afeta o card com a classe `.disappear`.

/* Estilo base do card */ .card { width: 250px; padding: 20px; box-shadow: 0 10px 20px rgba(0,0,0,0.15); transition: all 0.3s ease; } /* Efeito ao passar o mouse */ .card:hover { transform: translateY(2px); box-shadow: 0 2px 5px rgba(0,0,0,0.2); } /* A REGRA PRINCIPAL: */ /* Quando o #card-toggle estiver MARCADO (:checked), */ /* o .card.disappear que vem DEPOIS dele (+) */ /* ficará com opacidade 0. */ #card-toggle:checked + .card-container .card.disappear { opacity: 0; transform: scale(0.8); }
