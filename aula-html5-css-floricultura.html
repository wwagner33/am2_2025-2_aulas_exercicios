Cenário da Entrevista

Esta atividade partiu de um cenário fictício proposto pelo Prof. Wellington Sarmento, simulando uma entrevista de levantamento de requisitos para uma aplicação web de floricultura.

Contexto do Cliente

Durante a entrevista, o cliente descreveu suas necessidades:

  • Exibir produtos através de uma aplicação web
  • Apresentar produtos em cards com: nome, descrição, preço e ícone para adicionar ao carrinho
  • Permitir navegação intuitiva entre páginas
  • Oferecer processo de compra com seleção de quantidade, forma de pagamento e endereço
  • Fornecer feedback claro em caso de problemas
  • Permitir cancelamento a qualquer momento
  • Incluir botão "Home" para retornar à página inicial

Requisitos do Protótipo HTML5/CSS

Para atender às necessidades do cliente, definimos os seguintes requisitos:

  1. Estrutura semântica usando HTML5
  2. Layout responsivo que funcione em dispositivos móveis e desktop
  3. Sistema de navegação claro e intuitivo
  4. Apresentação visual atraente dos produtos
  5. Hierarquia de informação bem definida
  6. Fluxo de compra simplificado (para protótipo)
  7. Feedback visual para ações do usuário
  8. Tipografia legível e adequada ao tema

Planejamento do Design

Tipografia

  • Títulos: "Playfair Display" - elegante e sofisticada
  • Texto corrido: "Poppins" - legível e moderna

Navegação

  • Header fixo com logo e menu principal
  • Ícones de ação (carrinho, usuário, pesquisa)
  • Breadcrumbs visuais implícitos
  • Footer com informações de contato

Hierarquia de Informação

  1. Identidade da marca (header)
  2. Destaque principal (hero section)
  3. Produtos em grid
  4. Informações institucionais (footer)

Fluxo da Aplicação

  1. Página inicial → Lista de produtos
  2. Produto → Adicionar ao carrinho
  3. Carrinho → Finalizar compra (simulado)

Partes a serem Prototipadas

  • Página inicial com lista de produtos
  • Header com navegação
  • Footer institucional
  • Layout do carrinho (visualização básica)

Dica de Design

Utilize uma paleta de cores baseada na natureza para transmitir a essência da floricultura. Verdes, rosas suaves e amarelos funcionam bem neste contexto.

Implementação do Protótipo

Agora vamos implementar o protótipo seguindo os requisitos e planejamento definidos.

Atenção

Este é um protótipo estático que não inclui funcionalidades JavaScript avançadas. O foco é na estrutura HTML e estilização CSS.

Estrutura de Arquivos

Crie os seguintes arquivos em uma pasta do projeto:

  • index.html - Página principal
  • style.css - Estilos da aplicação
  • carrinho.html - Página do carrinho (opcional)

Principais Tags HTML5 Utilizadas

Estrutura Básica
  • <!DOCTYPE html>: Declara que este é um documento HTML5
  • <html lang="pt-br">: Elemento raiz com atributo de idioma
  • <head>: Contém metadados sobre o documento
  • <body>: Contém todo o conteúdo visível da página
Metadados
  • <meta charset="UTF-8">: Define a codificação de caracteres
  • <meta name="viewport">: Configura a viewport para responsividade
  • <title>: Define o título da página
Elementos Semânticos
  • <header>: Cabeçalho da página ou seção
  • <nav>: Seção de navegação
  • <section>: Seção temática de conteúdo
  • <footer>: Rodapé da página ou seção
Conteúdo
  • <h1> a <h3>: Títulos com diferentes níveis de hierarquia
  • <p>: Parágrafo de texto
  • <a href="...">: Link para outra página ou recurso
  • <img src="..." alt="...">: Imagem com texto alternativo
  • <ul> e <li>: Lista não ordenada e seus itens
  • <button>: Botão clicável

Explicação Detalhada do CSS

Variáveis CSS

As variáveis CSS (definidas em :root) permitem reutilizar valores em todo o código, facilitando a manutenção e garantindo consistência visual.

Box-sizing

O valor border-box faz com que padding e border sejam incluídos na largura e altura totais dos elementos, facilitando o cálculo de dimensões.

Flexbox e Grid
  • display: flex: Cria um container flexível para layouts unidimensionais
  • display: grid: Cria um sistema de layout bidimensional (linhas e colunas)
  • justify-content e align-items: Controlam o alinhamento nos eixos principal e transversal
Transições

A propriedade transition cria animações suaves entre estados (como hover), melhorando a experiência do usuário.

Media Queries

As regras @media aplicam estilos específicos baseados no tamanho da tela, garantindo que o layout seja responsivo.

Posicionamento
  • position: sticky: Fixa um elemento quando atinge uma determinada posição durante a rolagem
  • position: relative e absolute: Permitem posicionamento preciso de elementos
